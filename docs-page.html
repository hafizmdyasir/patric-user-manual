Introduction Last updated: 2024-08-08

PaTriC stands for Particle Tracker in C++. It is a fully relativistic charged particle tracking code that can simulate the motion of any number of targets in specified electric and magnetic fields.

PaTriC has been designed keeping students and researchers in mind. It utilizes the isolated particle model and thus, is suitable for generating a tractable idea of the response of systems to a given field configuration.

PaTriC was developed at the Indian Institute of Technology by Mohammad Yasir under the supervision of Professor Vikrant Saxena.

What PaTriC Does?

PaTriC can simulate the motion of any number of targets in specified electric and magnetic fields, following the isolated particle model. It ignores inter-particle interactions and simulates how a charged particle will move.

PaTriC is designed to be highly flexible and easy to use, allowing users to customize every aspect of the simulation and output. When you set up the simulation, you instruct PaTriC about the time-step, the integrator, the number of particles, etc. PaTriC uses the specified integration algorithm to solve the equation of motion for the target particle using the Lorentz Force as the driving force.

Install and Run

PaTriC does not ship in the form of executable binaries. Instead, you must download the source code and compile it on your system. Owing to certain features utilizing Unix-specific headers, PaTriC is only available on Unix operating systems. In this section, you will find everything you need to know about downloading, installing, and getting started with PaTriC.

Code Structure

PaTriC is primarily written in C++ but there are minor pieces of the code that utilize Python. To setup a simulation, you create an input script; a file that tells PaTriC all it needs to know about the simulation. This file is passed to the patric executable as the first argument. Once an input script is ready, you can run PaTriC by using the terminal command patric path/to/input.py

This input file is written in Python and contains the following information:

  1. Essential information about the simulation like time-step, number of iterations, etc.
  2. Information about your target particle like charge, mass, initial conditions, etc.
  3. Information about the electric and magnetic fields.
  4. Information about the program output.

Prerequisites

To get started, you will need the following:

  1. A C++ compiler (e.g., g++, clang++)
  2. Python 3.x
  3. The pypatric package. Get it here.

Furthermore, it is assumed that you have, at the very least, a basic understanding of how simulations work, python programming, and some knowledge about the terminal in the linux operating system.

Download & Install

You can grab the latest version of PaTriC from its GitHub releases page. Use the big green button in the top-right corner to get there. The release is provided as a zipped archive. When you extract it, you will end up with a folder named PaTriC whose structure is as follows:

	| PaTriC
	| ---- README.md
	| ---- LICENSE.txt
	| ---- compile.sh
	|
	| ---- source/
	|	/--- several .cpp files
	|
	|	/--- headers/
	|		/---- several .h files
	|
	| ---- runtime/
	|	/---- deck_tools.py
	|	/---- pyparser.py
	|	/---- pyserver.py
	|
	|	/---- pypatric/
	|		/---- several .py files
	|
	| ---- plotters/
	|	/---- 2DPlot.py
	|	/---- 3DPlot.py
	|	/---- utils.py

All you need to worry about here is the compile.shscript. It will take care of everything for you. Fire up a terminal in the extracted PaTriC folder and make this script executable using the command
$ chmod +x ./compile.sh.

Now, run this script using ./compile.sh and follow the on-screen instructions. If all goes well, you will find an executable named patric in the runtime folder. Add this folder to your system path and you're ready to go.

Run PaTriC

If you have added the patric executable to your system path, running it is as simple as firing up terminal and typing patric. We strongly recommend this approach since this makes it easy to run PaTriC from anywhere, and ensures that the required python scripts for full functionality of the program are available for the program to use.

The Input Script

As stated before, the input script for PaTriC is written in Python, and the path to this script is passed to the PaTriC executable as the first argument. The program then parses it and displays the parsed information before asking whether to proceed. This is a good point to recheck whether you have provided all parameters correctly or not.

In the following sections, we will understand the nuances of the input script and learn how to setup a simulation.

Overview

The input script consists of four python dict objects named Control, Particle, Fields, and Output. Each of these dictionaries serves a specific purpose and contains key-value pairs where the keys are special keywords defined inside the PaTriC source code.

Since it is understandbly difficult to remember, recall, and type the keywords each time you set up a simulation, we have created a package called pypatric that makes your job easier. This package also validates your parameters to ensure nothing unexpected happens during program execution.

The four dictionaries are defined using four functions from the pypatric.deck_helper module. Thus, every time you want to setup a simulation, start by creating a python file and importing this module.

The Control Dict

The purpose of the Control dictionary is to inform PaTriC about the control of the simulation. This includes the time-step for each iteration, the number of iterations before the program stops, the integration algorithm, and an optional message display frequency.

The calling signature for the function to define the Control dictionary is createControl(timeStep: float, numiterations: int, integrator: int, readoutFreq: int).

Time Step

In each iteration of the program, the particle advances by a time-step dt across its trajectory in the phase space. This value is passed as the first argument to the createControl function.

The choice of the time-step depends on the user and the system being studied. There is no fixed formula that will give the best results. Typically, finer step-sizes give more accurate results, but require a large number of iterations, which increases computation time and memory consumption. Further, the choice of the integrator also affects the accuracy of the output.

For most cases, we recommend a step-size of at least T/10, where T is the time-period of the fastest process in the system. For increased accuracy, one can use T/20, T/50, or even T/100, but the latter is overkill.

Number of Iterations

This is passed as the second argument and defines how many steps the program will run for. Any process happening within the system will require at least T/dt iterations to be fully resolved.

Integrator

PaTriC has support for three explicit, second-order integration algorithms. These are the Boris push, the Vay integrator, and the Higuera-Cary push. Use predefined keywords BORIS, VAY, and HIGUERA_CARY to choose which integrator you need.

The performance of each of these integrators has been well-studied before and the Boris push usually suffices. The Vay integrator comes at a higher computation cost but can yield better cross-field drifts, while the Higuera-Cary integrator has the best phase performance.

Stdout Frequency

You can instruct PaTriC to display a short message every n iterations. n is passed as the fourth argument to the function and can be 0 if you want to disable this functionality. This does not affect the performance or accuracy of the simulation.

The Particle Dict

The Particle dictionary is where you define the target of your simulation. PaTriC provides the functionality to simulate a large number of charged particles in succession, each with a given initial condition. This is useful for studying, say, a distribution of particles in a given configuration. Note that regardless of how many particles you simulate, they will be studied separately and remain uninfluenced by each other.

The Particle dictionary is created using the signature createParticles(mass: float, charge: float, particleCount: int, initialPosition: Tuple, initialVelocity: Tuple). The charge and mass are assigned commonly to each of the particles, while the initial positions and velocities can either be assigned separately, or drawn from a uniform distribution, or generated randomly.

Number of Particles

This is an integer parameter and is limited only by the configuration of your machine. It is theoretically possible to study as many as 106 particles, but that is an unrealistic number.

The Charge and Mass of the Particles

The charge is provided in units of the fundamental charge, e = 1.6E19 C. It can be any signed floating point number. The same charge is assigned to all the particles in the simulation.

Mass is provided in units of the mass of electron me = 9.1E-31 kg. It is also given to all the particles. This quantity must be a positive float.

Initial Position

The initial position argument must be provided as a python tuple. You may either provide a separate initial position for each of the particles using the Static class defined in pypatric.classes module, or draw it uniformly between a given range using Uniform, or generate randomly using Random. The explanation for each of these classes is given in the section that explains the pypatric module.

Initial Velocity

The initial velocity is also provided as a tuple containing either Static type objects, or a single Uniform or Maxwellian object. The former gives uniformly distributed velocities between the given range, while the latter samples from a Maxwellian distribution centered at 0 with FWHM given by the temperature provided as an argument.

Warning

In python, tuples are typically defined using a pair of parantheses (). However, when you request Uniform or Random or Maxwellian placement, there will be only a single item in the tuple. In such a case, the code (Maxwellian(temp)) will be considered the same as Maxwellian(temp), i.e., the tuple won't be created. To get around this, always place a guard empty element in the tuple.
Use (Maxwellian(temp), ).

Tip

The Maxwellian class constructor takes its input in kelvins. If you wish to work in electron-volts, a helper function called eVToKelvin has been defined in the pypatric package for your convenience.

The Fields Dict

This is the part where you define the electric and magnetic fields that will be used to evolve the system. The syntax is fairly simple. Call the function createFields(electricFields: Tuple | Callable, magneticFields: Tuple | Callable).

The tuple must contain either a number of objects of pre-defined classes (see PyPaTriC - Classes), or a function named getEField or getBField taking the arguments x,y,z,t, and returning the three components of the electric and magnetic fields as a tuple.

In the former case, the value of the field will be a superposition of the values calculated by each of the objects passed to the tuple. In the latter case, the tuple must only contain one entry: the function with the signature as stated just now. A python interpreter will be used to call this function at each iteration. This is slightly more expensive.

The Output Dict

The function of the Output dictionary is to inform PaTriC how to dump the data from the simulation. This includes the file name for output, the file path, any optional data you want to put in the info file, and the variables to dump.

The Output dictionary is created using the syntax createOutput(fileName: str, filePath: str, fileHeaders: Tuple[str, ...], dumpInfoFile: bool, dumpVariables: Tuple[str, ...]). A description is as follows:

File Name and Path

The fileName argument is a string representing the name of the output file. It should not include any extension. PaTriC will automatically assign .dat extension to the output files. The filePath argument is a string representing the directory path where the output file will be saved. If the directory does not exist, it will be created automatically.

fileHeaders Argument

This is a tuple of strings that will be written to an info file dumped alongside the output files, containing basic information about the simulation run. Note that regardless of the number of particles, only a single info file is created and it has the same name as provided via fileName.

dumpInfoFile Flag

If you want, you can disable the creation of the info file by passing False to this parameter. The default value is True and thus, info file will be created by default.

dumpVariables Argument

This is a tuple of strings that lists the variables you want to dump. Supported values inside this tuple are as follows:

  • POS_(X,Y,Z): The (x,y,z)-component of position.
  • VEL_(X,Y,Z): The (x,y,z)-component of velocity.
  • E_FIELD_(X,Y,Z): The (x,y,z)-component of the electric field calculated at each time instant.
  • B_FIELD_(X,Y,Z): The (x,y,z)-component of the magnetic field calculated at each time instant
  • GAMMA: The relativistic factor γ at each time instant.

Analyzing Output

The simulation data from PaTriC is saved as simple .dat files that are human readable and compatible with almost all plotting software. The plotters that ship with PaTriC are under construction and for now, your best bet is to use the numpy.loadtxt and use matplotlib to plot the data as per your convenience.

Data Structure

The output data file contains tabspace separated columns, with each column corresponding to a particular variable, and each row corresponding to a particular time-step. All data is dumped in SI units, in scientific format, with precision of 12 decimal places.

For each particle you load in the simulation, a separate output file will be created, and numbers starting with 0 will be appended at the end of the file name, corresponding to each particle.

The first row of is a header containing the description of each of the columns. The order of these columns is fixed and if a particular variable is not requested in the dumpVariables tuple in the Output dictionary, its column is not created in the output file.

Order of Columns in Output File
Column Variable Dumped Units
1 Time Always Seconds
2,3,4 Position (X,Y,Z) On request Metres
5,6,7 Velocity (X,Y,Z) On request Metres per second
8 Relativistic Factor γ On request Dimensionless
9,10,11 Electric Field (X,Y,Z) On request Volts per metre
12,13,14 Magnetic Field (X,Y,Z) On request Tesla

Note

Since the electric and magnetic fields are calculated at the beginning of each iteration, the very last iteration does not have information about them. Thus, the last row for electric and magnetic fields will contain NaN.

The Info File

This is a plain text file named as <fileName>.info. The strings you provide in the fileHeaders tuple will be written at the top of this info file, with each string in a new line.

The info file also contains the the total time taken (tt) to run the program, the time-step used (dt), and the number of tierations (ni). And finally, the target information including charge, mass, number of particles, and initial conditions for each of the particles are also dumped in this file. This can come in handy when analyzing the output.
